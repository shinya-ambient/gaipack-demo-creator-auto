# アーキテクチャルール

## ディレクトリ構成
```
demo-app/src/
├── components/    # 再利用可能なUIコンポーネント
├── pages/         # 画面（ルーティング対象）
├── types/         # TypeScript型定義
├── data/          # モックデータ
├── hooks/         # カスタムフック
├── utils/         # ユーティリティ関数
└── index.css      # グローバルCSS（デザイントークン）
```

## 制約
- コンポーネントは1ファイル200行以内
- any型の使用禁止（unknownを使う）
- console.logをコミットしない
- インラインスタイルよりTailwind CSSクラスを優先
- 外部APIキーはハードコードしない（.envを使う）
- モックデータはリアリティのある日本語（テスト太郎NG）
- 全インタラクティブ要素にホバー/フォーカスのフィードバック
- ローディング状態はスケルトンUIかスピナーで表現
